# "Interpretation Rules.txt"

# Adobe After Effects 自动素材解释

# 允许空行，并且以“#”开头的行是注释

# 硬性规则的基本格式: {匹配要求} = {设置解释}
# 软性规则的基本格式: {匹配要求} ~ {设置解释}

# + 对于匹配的每个规则，设置了解释；双方都允许通配符（“*”）；
#	通配符匹配将匹配任何内容，通配符设置不会设置任何内容
# + 允许匹配多个规则，后面的匹配项将覆盖前面的匹配项
# + 按顺序扫描所有规则，每行一个规则
# + 在像素长宽比、Alpha 信息和字段标签之后应用规则
#	从文件中读取信息

# + 软性规则不会覆盖由文件
#	根据以前的规则设置格式或进行设置

# 宽度、高度、帧速率、文件类型、编解码器 =
#	像素长宽比/名称、场顺序、匹配帧速率、Alpha 解释、icc 配置文件代码明确指定的值
#	或配置文件描述（一个或其他）、线性光

# 宽度和高度是整数
# 帧速率是十进制数
# 文件类型是引号中的四个字符 (FourCC)
# 编解码器（仅适用于 QT 和 AVI）也是 FourCC

# 用于查找特定类型的素材的文件类型和/或编解码器 FourCC，
#	按住 opt 键并单击 (Mac) 或按住 alt 键并单击 (Windows) 项目窗口中的素材；
#	FourCC 代码将通过缩览图显示在文字的最后一行中

# 像素长宽比是整数的比例（例如 10/11）
#	可以选择后跟该比例的名称（例如 10/11/“我的长宽比”）
# 在 After Effects CS4 (v9) 中，我们修改了多种常用格式的像素长宽比:
#	从 648/720 (0.9) 到 10.0/11.0（大约 0.9091）的 D1/DV NTSC
#	从 6/5 (1.2) 到 40/33（大约 1.2121）的 D1/DV NTSC 宽银幕
#	从 768/720 (1.07) 到 768/702（大约 1.0940）的 D1/DV PAL
#	从 64/45 (1.42) 到 1024.0/702.0（大约 1.4587）的 D1/DV PAL 宽银幕

# 场顺序是单个字符（F=帧、U=高场或 L=低场）
# 匹配帧速率是十进制数
# Alpha 解释:（I=忽略、S=直接、P=预乘/黑色、W=预乘/白色）
#	为不带 Alpha 通道的素材忽略了 Alpha 解释

# ICC 配置文件由是 FourCC 的配置文件代码指定或由
#	配置文件描述是用引号括起来的加上引号最多 255 个字符的字符串

# 受支持的配置文件代码是:
#	- 工作空间配置文件:“wRGB”
#	- 嵌入配置文件:“embe”
#	- 保留 RGB:“pRGB”（这准确执行“解释素材”对话框中的“保留 RGB”复选框的功能）
#	- sRGB: "sRGB"
#	- Adobe RGB:“AS98”
#	- Kodak 5218 胶片配置文件:“ginp”
#	- SDTV NTSC:“r6nf”（SMPTE RP 145 原色和 ITU-R Rec. 709 TRC。黑色编码为 0，白色编码为 1。）
#	- SDTV NTSC 16-235:“r6ns”（SMPTE RP 145 原色和 ITU-R Rec. 709 TRC。黑色编码为 16/255，白色编码为 235/255。）
#	- SDTV PAL:“r6pf”（EBU Tech. 3213 原色和 ITU-R Rec. 709 TRC。黑色编码为 0，白色编码为 1。）
#	- SDTV PAL 16-235:“r6ps”（EBU Tech. 3213 原色和 ITU-R Rec. 709 TRC。黑色编码为 16/255，白色编码为 235/255。）
#	- HDTV (Rec. 709):“r7hf”（ITU-R Rec. 709 原色和 TRC。黑色编码为 0，白色编码为 1。）
#	- HDTV (Rec. 709) 16-235:“r7hs”（ITU-R Rec. 709 原色和 TRC。黑色编码为 16/255，白色编码为 235/255。）

# 配置文件描述可在 UI（项目窗口缩览图信息）中或在
# ColorSync 之类的颜色实用工具中查找，或者由用户在创建自定义配置文件时定义

# 线性光是整数（0=关闭，1=打开，2=对 32 bpc 素材打开）
